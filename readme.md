# 任务管理助手

本程序使用C#开发，用于常驻桌面，支持个人及敏捷团队管理成员的工作待办事项。它通过重要紧急四象限帮助用户聚焦关键任务，并提供角色化任务分配与视图。

用于时刻提醒用户聚焦于重要且紧急的工作，减少工作中的干扰

## 功能

1. 任务分类 (Task Classification)

   任务根据其“重要性” (Importance) 和“紧急性” (Urgency) 被归类到四象限中（重要且紧急、重要不紧急、不重要但紧急、不重要不紧急）。
   这些属性的设置可以由用户手动指定，或通过LLM（大语言模型）辅助分析任务描述自动生成建议。
   任务现在还包含 `assignedRole` 字段，用于团队任务分配。

   todo LLM根据用户的描述，自动分类任务优先级的功能仍在开发中。

2. 团队任务管理 (Team Task Management)
    *   **角色定义与切换 (Role Definition & Switching):** 支持预定义角色（如经理、开发、QA、设计）。用户可以通过主界面的下拉菜单切换当前查看的角色。
    *   **任务分配 (Task Assignment):** 用户在创建新任务时，可以将任务分配给特定角色或标记为“未分配”。 （注：目前版本中，特定角色的分配主要体过UI选择，而“经理”角色理论上应能分配给任何角色，具体分配流程和权限可能依团队实践调整。）
    *   **角色化视图 (Role-Based Views):** 每个角色登录后（通过角色切换模拟），将主要看到分配给自己的任务。“经理”角色还可以看到未分配（"Unassigned"）的任务。
    *   **状态同步 (Status Synchronization):** 任务状态的变更（如完成、结果更新，尽管目前UI中对这些字段的直接编辑有限）会保存到共享的 `tasks.csv` 文件。团队成员通过“刷新”按钮或重新加载程序可以获取最新状态。

3. 任务提醒

   LLM辅助的智能提醒功能，例如针对长期未完成任务的分析和建议，目前已集成基础版本，会针对超过两周未修改的任务生成提醒和建议（输出到控制台）。
   todo 完整的用户界面集成和更高级的提醒机制仍在规划中。

4. 任务存储 (Task Storage)
    *   **数据存储 (Data Storage):** 所有任务现存储于程序 `data` 目录下的单一 `tasks.csv` 文件中，取代了旧版的四象限独立文件。该文件包含任务描述、优先级（重要性、紧急性）、分配角色、状态（是否活动）、创建日期、最后修改日期、以及任务结果等信息。

5. 任务管理 (Task Management)

   基本的任务增、删、查（通过角色视图）功能。通过点击任务旁的"X"按钮删除任务。通过主界面的“Add New Task”按钮添加新任务。
